# vibe/mcp-server

The **framework-facing MCP server** that lets any Model Context Protocol client
(Claude Code, Cursor, Kilo, …) **operate, build, and generate Vibe** — and,
together with the meta "Vibe engineer" agent, makes Vibe *itself* designed,
operated, and generated by AI agents.

## What it exposes

Two tool namespaces, plus resources and prompts:

- `vibe_runtime_*` — operate a running Vibe system: run the agent loop
  (`vibe_runtime_ask` / `vibe_runtime_run`), seed and inspect the tool
  registry (`vibe_runtime_seed_tools`, `vibe_runtime_list_tools`,
  `vibe_runtime_get_tool`), and read live status (`vibe_runtime_status`).
- `vibe_dev_*` — build and generate Vibe: scaffold packages/agents
  (`vibe_dev_scaffold_package`, `vibe_dev_scaffold_agent`), run pnpm
  scripts (`vibe_dev_run`, `vibe_dev_check`), list the package graph
  (`vibe_dev_info`), and drive the meta agent (`vibe_dev_engineer_run`).
- **Resources**: `vibe://packages`, `vibe://model-catalog`, `vibe://grammar`,
  `vibe://runtime/status`.
- **Prompts**: `vibe/design-agent`, `vibe/debug-run`, `vibe/add-tool`,
  `vibe/scaffold-package`.

## Run it

```bash
pnpm -F vibe/mcp-server build
# stdio (default — what coding agents connect to):
node packages/mcp-server/dist/server.js
# or via the inspector:
pnpm -F vibe/mcp-server inspect

# streamable HTTP:
VIBE_MCP_TRANSPORT=http VIBE_MCP_PORT=3000 node packages/mcp-server/dist/server.js
```

Set `ANTHROPIC_API_KEY` so the agent-running tools can reach a provider.

## Design note

Every capability is defined **once** as an `McpTool` in `src/tools/*`. The
server registers it for MCP, and `src/agent/vibe-engineer.ts` wraps the same
definitions as `vibe/tools` `Tool`s for the meta agent — one schema, two
consumers, mirroring Vibe's `defineTool` philosophy.
